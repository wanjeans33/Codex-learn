开始学习

Codex 关键功能

1) Plan Mode（计划模式）

将复杂任务拆解成可执行步骤（如：调研 → 改代码 → 测试 → 提交），并在执行前给你可审阅的计划。

  • 适合多阶段任务，减少“直接改错方向”的风险。
  • 便于团队协作：先对齐思路，再执行。
  • 支持阶段性更新，随时修订计划。

2) Custom Agent（自定义代理）

你可以定义自己的执行偏好和规则，例如代码风格、测试标准、提交规范，让代理行为更贴近团队流程。

  • 固定输出格式（如 PR 模板、日志模板）。
  • 固定工具链（测试、构建、检查命令）。
  • 减少重复指令，提高稳定性。

3) Agent Skill（技能体系）

Skill 是可复用的能力包（带流程、脚本、模板）。针对某类任务触发后，代理会按技能说明执行。

  • 适合标准化高频任务（如发布流程、文档生成）。
  • 支持“按需加载”，避免一次塞入过多上下文。
  • 可持续迭代：技能本身可版本化管理。

快速上手流程（推荐）

  1. 明确目标：例如“修复支付回调 bug 并补测试”。
  2. 用 Plan Mode 拆任务并确认里程碑。
  3. 调用或配置 Custom Agent（约束代码风格和提交要求）。
  4. 在合适步骤触发 Agent Skill（如自动生成变更日志）。
  5. 运行测试与检查，提交并生成 PR。

对比：Claude Code vs GitHub Copilot vs Codex

能力维度 Claude Code GitHub Copilot Codex
计划驱动（Plan Mode） 有任务规划能力，模式呈现依产品形态而定 以编辑器内补全/对话为主，计划流程相对轻量 强调可审阅的分步计划与执行状态
自定义代理（Custom Agent） 可通过提示与配置形成偏好 可借助指令与仓库上下文增强 支持更工程化的代理行为约束
技能机制（Agent Skill） 可通过工作流封装实现部分复用 依赖扩展生态与提示模板 原生强调“技能化”复用流程与脚本
终端/仓库执行闭环 具备较强代码与命令协作能力 IDE 场景强，端到端自动执行视配置而定 面向代码库任务闭环（改、测、提）
适合人群 重视高质量对话与分析的开发者 日常编码补全与 IDE 内提效用户 希望把 AI 纳入工程流程的团队/个人

说明：不同产品能力会快速演进，具体功能以各平台最新官方文档为准。
